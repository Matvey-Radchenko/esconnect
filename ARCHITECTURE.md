# Архитектура ESConnect v2.0.0

## Обзор системы

ESConnect - это автоматизированная система для подключения к Endpoint Security VPN на macOS с локальным HTTP API для приема 2FA кодов. Система написана на Rust и состоит из нескольких модулей, каждый из которых отвечает за определенную функциональность.

## Общая архитектура

│                 │    │   (daemon.rs)   │    │                 │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │                       │

## Компоненты системы

### 1. CLI Interface (main.rs)

**Ответственность:**

- Парсинг командной строки с помощью clap
- Диспетчеризация команд в соответствующие обработчики
- Управление жизненным циклом приложения

**Ключевые структуры:**

- `Cli` - структура для парсинга аргументов
- `Commands` - enum с доступными командами
- `SetupAction` - enum для действий настройки

**Поток выполнения:**

```
main() → env_logger::init() → parse CLI → match command → execute handler
```

### 2. Background Daemon (daemon.rs)

**Ответственность:**

- Запуск и управление HTTP сервером
- Graceful shutdown через сигналы ОС
- Инициализация всех компонентов системы

**Ключевые компоненты:**

- `Daemon` struct с полем `running: Arc<AtomicBool>`
- Запуск асинхронного HTTP сервера через `start_server()`
- Обработка сигналов завершения (Ctrl+C)

**Алгоритм работы:**

```
1. Инициализация конфигурации
2. Создание KeychainManager и VpnAutomator
3. Запуск HTTP сервера на указанном порту
4. Ожидание завершения (graceful shutdown)
```

### 3. HTTP Server (server.rs)

**Ответственность:**

- Локальный HTTP сервер для приема 2FA кодов
- Аутентификация по токену (X-Auth-Token header)
- IP whitelist для безопасности
- Валидация формата кодов (6 цифр)

**Ключевые структуры:**

- `ServerState` - состояние сервера с конфигом и компонентами
- `TokenRequest` - структура для JSON запроса {"code": "123456"}
- `TokenResponse` - структура ответа сервера

**Эндпоинты:**

- `POST /token` - прием 2FA кода с аутентификацией
- Проверки: IP whitelist, auth token, формат кода
- При успешной валидации вызывает VpnAutomator::connect()

### 4. VPN Automation (vpn.rs)

**Ответственность:**

- Автоматизация GUI взаимодействия с VPN клиентом
- Симуляция кликов мыши и ввода клавиатуры
- Управление задержками для надежности

**Ключевые компоненты:**

- `VpnAutomator` struct с `driver` и `config`
- Последовательность действий для подключения

**Алгоритм подключения:**

```
1. Клик по иконке VPN (позиция из конфига)
2. Задержка menu_open
3. Навигация стрелкой вниз (key code 125)
4. Задержка menu_navigation
5. Нажатие Enter (key code 36)
6. Задержка dialog_wait
7. Ввод кода через буфер обмена
8. Задержка input_delay
9. Нажатие Enter
10. Задержка connection_wait
11. Ввод пароля через буфер обмена
12. Нажатие Enter
```

### 5. RPA Module (rpa/)

**Ответственность:**

- Абстракция GUI автоматизации
- Кроссплатформенный интерфейс (трейт GuiDriver)
- Реализация для macOS

#### driver.rs

- `GuiDriver` trait с методами: click, type_text, press_key, get_mouse_position

#### macos_driver.rs

- `MacOsDriver` struct реализующий GuiDriver
- Использование `cliclick` для кликов и позиционирования
- `osascript` для нажатий клавиш
- Буфер обмена для надежного ввода текста

### 6. Configuration Management (config.rs)

**Ответственность:**

- Управление конфигурацией приложения
- Сериализация/десериализация JSON
- Использование стандартных директорий ОС

**Ключевые структуры:**

- `Config` - основная структура конфигурации
- `ConfigManager` - менеджер с путем к файлу
- Вложенные структуры: ServerConfig, VpnConfig, SystemConfig, DelaysConfig

**Путь к конфигу:**

```rust
let proj_dirs = ProjectDirs::from("com", "esconnect", "esconnect")?;
let config_dir = proj_dirs.config_dir(); // ~/Library/Application Support/com.esconnect.esconnect/
let config_path = config_dir.join("config.json");
```

### 7. Security (security.rs)

**Ответственность:**

- Безопасное хранение чувствительных данных
- Интеграция с системным Keychain

**Ключевые компоненты:**

- `KeychainManager` struct
- Использование `keyring` crate
- Сервис: "esconnect"

**Методы:**

- `set_password(key, password)` - сохранение в Keychain
- `get_password(key)` - извлечение из Keychain
- `delete_password(key)` - удаление из Keychain

### 8. UI Library (ui.rs)

**Ответственность:**

- Красивый консольный интерфейс
- Интерактивные запросы пользователя
- Отображение прогресса и статусов

**Ключевые компоненты:**

- `Ui` struct с статическими методами
- Использование `colored` для цветов
- `indicatif` для спиннеров
- `inquire` для интерактивных запросов

### 9. Utilities (utils.rs)

**Ответственность:**

- Вспомогательные функции
- Проверка статуса VPN

**Функции:**

- `check_vpn_active()` - проверка через `scutil --nc list`

## Потоки данных

### Настройка системы

```
CLI (setup) → Ui::ask_* → KeychainManager::set_password
                    → MacOsDriver::get_mouse_position → ConfigManager::save
```

### Работа демона

```
HTTP Request → Server::handle_token → validate request → VpnAutomator::connect
                                      → KeychainManager::get_password
```

### VPN подключение

```
VpnAutomator::connect → MacOsDriver::click → MacOsDriver::press_key
                      → MacOsDriver::type_text → delays via thread::sleep
```

## Безопасность

### Хранение данных

- Пароли и токены: системный Keychain (шифрование ОС)
- Конфигурация: JSON файл с правами 600
- Логи: /tmp/esconnect.log (временные файлы)

### Сетевое взаимодействие

- Локальный HTTP сервер: прием POST запросов на порту 3000
- IP whitelist: ограничение доступа по IP адресу
- Auth token: проверка X-Auth-Token заголовка
- Валидация: проверка формата 2FA кодов

### GUI автоматизация

- Буфер обмена: временное использование с восстановлением
- Задержки: предотвращение обнаружения как бота
- Позиционирование: пользовательская настройка координат

## Обработка ошибок

### Уровни обработки

1. **Модульный уровень**: `anyhow::Result` для всех операций
2. **Daemon уровень**: экспоненциальная задержка при ошибках API
3. **Системный уровень**: graceful shutdown через Ctrl+C

### Стратегии восстановления

- **HTTP сервер**: автоматический перезапуск при ошибках
- **VPN подключение**: повторные попытки с теми же данными
- **Конфигурация**: значения по умолчанию при отсутствии файла
- **Keychain**: graceful handling отсутствия данных

## Производительность

### Оптимизации

- **Async/await**: неблокирующие HTTP операции
- **Логирование**: только необходимые сообщения
- **Память**: минимальный footprint благодаря Rust
- **Безопасность**: IP whitelist и token аутентификация

### Ресурсы

- **CPU**: низкая нагрузка в фоне
- **Память**: ~10-20MB resident
- **Сеть**: локальный трафик (HTTP на localhost)
- **Диск**: конфиг ~1KB, логи ротируются ОС

## Расширение системы

### Добавление новой платформы

1. Реализовать `GuiDriver` для новой ОС
2. Обновить `VpnAutomator` для использования нового драйвера
3. Адаптировать пути конфигурации в `ConfigManager`

### Добавление нового источника кодов

1. Создать новый модуль для приема кодов (HTTP, WebSocket, etc.)
2. Реализовать интерфейс для получения кодов
3. Интегрировать в `server.rs` или создать новый модуль

### Добавление новых команд

1. Расширить `Commands` enum в `main.rs`
2. Добавить обработчик в `match` блоке
3. Реализовать логику в отдельной функции

## Мониторинг и отладка

### Логирование

- **Уровни**: ERROR, WARN, INFO, DEBUG
- **Формат**: timestamp level module message
- **Вывод**: stderr (перенаправляется в файл при daemon режиме)

### Метрики

- Количество обработанных кодов
- Время отклика HTTP сервера
- Успешность VPN подключений
- Частота ошибок аутентификации
- IP адреса отправителей запросов

### Тестирование

- Unit тесты для каждого модуля
- Integration тесты для полного цикла
- GUI тесты с моками драйвера
